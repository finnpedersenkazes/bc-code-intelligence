# ALDoc for Reference Documentation Generation - Code Samples

## Installation Commands

```powershell
# Install .NET 6.0 SDK (if not present)
winget install Microsoft.DotNet.SDK.6

# Install DocFx 2.70.0 - MUST be this exact version
dotnet tool install -g docfx --version 2.70.0

# Verify DocFx version
docfx --version
# Expected output: 2.70.0+...
```

## Finding ALDoc Path

```powershell
# ALDoc ships with AL Language VS Code extension
# Typical paths:

# Windows
$aldocPath = "$env:USERPROFILE\.vscode\extensions\ms-dynamics-smb.al-*\bin\win32\aldoc.exe"

# macOS
$aldocPath = "~/.vscode/extensions/ms-dynamics-smb.al-*/bin/darwin/aldoc"

# Linux
$aldocPath = "~/.vscode/extensions/ms-dynamics-smb.al-*/bin/linux/aldoc"

# Find exact path
Get-ChildItem -Path "$env:USERPROFILE\.vscode\extensions" -Filter "ms-dynamics-smb.al-*" |
    Select-Object -Last 1 |
    ForEach-Object { Join-Path $_.FullName "bin\win32\aldoc.exe" }
```

## ALDoc Init Command

```powershell
# Initialize documentation repository
aldoc init `
    -o "C:\Docs\MyAppDocs" `
    -t "C:\Apps\MyApp.app" `
    -c "C:\Dependencies"

# Parameters:
# -o : Output directory for documentation project
# -t : Target app file to document
# -c : Cache folder with dependent .app files
```

## ALDoc Build Command - Single App

```powershell
# Generate reference files for one app
aldoc build `
    -o "C:\Docs\MyAppDocs" `
    -s "C:\Apps\MyApp.app" `
    -c "C:\Dependencies"

# Parameters:
# -o : Output directory (same as init)
# -s : Source app file being documented
# -c : Dependencies for cross-references
```

## Full Workflow Script

```powershell
# Complete ALDoc documentation workflow
param(
    [string]$AppPath = "C:\Apps\MyApp.app",
    [string]$OutputPath = "C:\Docs\MyAppDocs",
    [string]$DependencyPath = "C:\Dependencies"
)

# Find ALDoc
$alExtension = Get-ChildItem "$env:USERPROFILE\.vscode\extensions\ms-dynamics-smb.al-*" |
    Sort-Object Name | Select-Object -Last 1
$aldoc = Join-Path $alExtension.FullName "bin\win32\aldoc.exe"

# Step 1: Initialize
Write-Host "Initializing documentation repository..."
& $aldoc init -o $OutputPath -t $AppPath -c $DependencyPath

# Step 2: Build reference content
Write-Host "Generating reference documentation..."
& $aldoc build -o $OutputPath -s $AppPath -c $DependencyPath

# Step 3: Build HTML site
Write-Host "Building HTML site..."
Push-Location $OutputPath
docfx build docfx.json

# Step 4: Serve for preview
Write-Host "Starting local server at http://localhost:8080..."
docfx serve _site
Pop-Location
```

```

├── docfx.json
├── toc.yml
├── index.md
├── reference/           # Generated by aldoc
│   └── ...

    ├── getting-started.md
    ├── concepts/
    │   └── architecture.md
    └── tutorials/
        └── first-steps.md
```

```yaml
# docfx.json - Add overwrites to build
{
  "build": {
    "content": [
      { "files": ["**/*.yml"], "src": "reference" },
      { "files": ["**/*.md"], "src": "overwrites" }
    ]
  }
}
```

## WRONG: Using Incompatible DocFx Version

```powershell
# WRONG - DocFx 2.71+ has breaking changes
dotnet tool install -g docfx --version 2.75.0

# Result: Documentation builds but renders incorrectly
# - Missing navigation
# - Broken cross-references
# - Style/layout issues
```

## CORRECT: Using Compatible DocFx Version

```powershell
# CORRECT - Use exactly version 2.70.0
dotnet tool install -g docfx --version 2.70.0

# If newer version installed, uninstall first
dotnet tool uninstall -g docfx
dotnet tool install -g docfx --version 2.70.0
```

## Multi-App Documentation Build Script

```powershell
# Build documentation for multiple related apps
$apps = @(
    "C:\Apps\BaseLibrary.app",
    "C:\Apps\CoreFeatures.app",
    "C:\Apps\MainApp.app"
)

$outputPath = "C:\Docs\MySolution"
$baseDeps = "C:\bcartifacts"

# Initialize with first app
& $aldoc init -o $outputPath -t $apps[0] -c $baseDeps

# Build each app, accumulating dependencies
$processedApps = @()
foreach ($app in $apps) {
    $cachePaths = @($baseDeps) + $processedApps
    $cacheArg = $cachePaths -join ";"

    Write-Host "Building: $app"
    & $aldoc build -o $outputPath -s $app -c $cacheArg

    $processedApps += $app
}

# Generate final HTML
docfx build "$outputPath\docfx.json"
```
